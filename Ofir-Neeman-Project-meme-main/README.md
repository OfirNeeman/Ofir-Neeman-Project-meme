# MemeMaster 🎬🃏

MemeMaster הוא משחק מולטיפלייר חברתי ואינטראקטיבי בזמן אמת, שבו שחקנים מתחרים ביניהם על יצירת המימים המצחיקים והטובים ביותר. המשחק משלב טכנולוגיות ווב מתקדמות יחד עם בינה מלאכותית (AI) שמשמשת כשופט האובייקטיבי והמשעשע של הסבב!

## 🚀 תכונות מרכזיות (Features)

- **משחק מולטיפלייר בזמן אמת:** הצטרפות ללובי, ניהול חדרים וסנכרון מלא בין כל השחקנים בעזרת Firebase Firestore.
- **שלבי משחק מוגדרים (Game Phases):**
  1. **Upload Phase:** העלאת תמונות בסיס למימים.
  2. **Captioning Phase:** כתיבת טקסטים (Captions) יצירתיים ומצחיקים על גבי התמונות שנבחרו.
  3. **Judging Phase (שיפוט AI):** שימוש ב-**Gemini AI** שמנתח את השילוב בין התמונה לטקסט, נותן ציון ומספק ביקורת משעשעת על כל מים.
  4. **Results & Winner Phase:** הצגת הדירוג הסופי והכרזת המנצח של הסבב.
- **ממשק משתמש מודרני:** עיצוב רספונסיבי, מהיר ונקי באמצעות React, TypeScript ו-Tailwind CSS.
- **ניהול תמונות וקבצים:** שרת Backend ייעודי ב-Python המטפל בהעלאת תמונות, תקשורת Sockets ואינטגרציה עם קבצים.

## 🛠️ טכנולוגיות (Tech Stack)

### Frontend
- **React (TypeScript)** & **Vite** (סביבת פיתוח מהירה וקלה)
- **Tailwind CSS** (עיצוב מודרני ורספונסיבי)
- **Firebase SDK** (Firestore לשמירת מצב המשחק, Authentication לניהול משתמשים)

### Backend & AI
- **Python Server** (ניהול Sockets, העלאת תמונות ועיבוד קבצים)
- **Gemini API** (מודל ה-AI המשמש לשפיטה והערכת המימים בזמן אמת)

## 🚀 הוראות הרצה ופרסום (Deployment & Running)

כדי להריץ את הפרויקט ולעדכן אותו, אנחנו מפעילים את ה-Frontend (הלקוח) ואת ה-Backend (השרת) בנפרד באופן הבא:

### 1. בנייה ופרסום ה-Frontend (Firebase)
פתחו את ה-CMD (שורת הפקודה) בתיקייה הראשית של הפרויקט והריצו את הפקודות הבאות כדי לבנות את האתר ולהעלות אותו לאוויר:
```bash
npm run build
firebase deploy

2. הרצת הלקוח המשלים / פרוקסי (Client Proxy)
כדי להפעיל את רכיב הלקוח הייעודי, פתחו והפעילו את הקובץ:

client_proxy.py

3. הרצת שרת ה-Backend (Python)
פתחו חלון CMD חדש בתוך תיקיית השרת (server) והריצו את השרת:

Bash
cd server
py server.py
